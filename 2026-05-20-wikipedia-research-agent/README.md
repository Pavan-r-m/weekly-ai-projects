# 🔍 Wikipedia Research Agent

An autonomous multi-step AI agent that researches any topic using Wikipedia as its knowledge source — **no API key required**.

Given a query like `"quantum computing"`, the agent:
1. **Searches** Wikipedia to find the best entry points
2. **Fetches** page summaries and extracts the most information-dense sentences
3. **Discovers** related sub-topics by following internal links
4. **Scores** candidate pages by relevance and queues the most promising ones
5. **Synthesizes** a structured, markdown-formatted research report

This project demonstrates core **agentic AI patterns** in clean, beginner-friendly Python:
- 🛠️ **Tool use** — agent calls Wikipedia REST API as a structured tool
- 🧠 **Memory** — `ResearchMemory` class accumulates facts across steps
- 🔄 **Iterative planning** — breadth-first exploration with configurable depth
- ✍️ **Synthesis** — all facts compiled into a coherent report

## How to Run

```bash
# Basic research (prints report to stdout)
python agent.py "quantum computing"

# Broader research: follow links 2 levels deep, fetch up to 8 pages
python agent.py "CRISPR gene editing" --depth 2 --pages 8

# Save the report to a Markdown file
python agent.py "climate change" --output report.md

# Suppress the agent trace, only show the final report
python agent.py "black holes" --quiet

# Optional: enhance report with GPT-4o-mini narrative synthesis
export OPENAI_API_KEY="sk-..."
python agent.py "machine learning" --llm --output ml_report.md
```

### CLI Options

| Flag | Default | Description |
|------|---------|-------------|
| `query` | *(required)* | Topic to research |
| `--depth` | `1` | Max link-follow depth (higher = broader) |
| `--pages` | `5` | Max Wikipedia pages to fetch |
| `--output` | stdout | Save report to Markdown file |
| `--llm` | off | Use GPT-4o-mini to write narrative synthesis |
| `--quiet` | off | Suppress agent trace, print only final report |

## How It Works

### Agentic Loop

```
┌─────────────────────────────────────────────────────┐
│                  ResearchAgent.run()                 │
│                                                      │
│  1. PLAN    search_wikipedia(query)                 │
│             → candidate page titles                 │
│                                                      │
│  2. ACT     fetch_page_summary(title)               │
│             extract_key_sentences(extract)          │
│             → facts stored in ResearchMemory        │
│                                                      │
│  3. OBSERVE fetch_related_links(title)              │
│             score_relevance(candidate, query_terms) │
│             → top 3 added to exploration queue      │
│                                                      │
│  4. REPEAT  until max_pages or queue empty          │
│                                                      │
│  5. SYNTHESIZE  _synthesize_report()                │
│             → structured Markdown report            │
└─────────────────────────────────────────────────────┘
```

### Key Sentence Extraction

Without a trained NLP model, the agent uses a scoring heuristic:
- **Capital-word density** — sentences with more proper nouns and named entities tend to be more informative
- **Length score** — medium-length sentences (15–40 words) capture complete ideas
- **Long-sentence penalty** — very long sentences (>60 words) are slightly penalized

### Relevance Scoring

Related Wikipedia links are ranked by **term overlap** with the original query:
```python
overlap = len(title_terms & query_terms) / len(query_terms)
```

This simple heuristic keeps the research focused without requiring an LLM.

## Optional LLM Enhancement

If you have an OpenAI API key, pass `--llm` to have GPT-4o-mini rewrite the
bullet-point report into flowing prose:

```bash
export OPENAI_API_KEY="sk-..."
python agent.py "reinforcement learning" --llm --output report.md
```

The stub gracefully degrades — if the key is absent, the structured report is
returned unchanged. This makes the core agent fully functional without any
external credentials.
